# 美术风格与换肤预留

> 当前定位：V02 美术生产基线。  
> 本文不讨论多主题并行落地，只规定当前 Animal Crossing-like cozy town 美术生产、Apple-like translucent glass UI、命名、逻辑 asset ID 和验收方式。

## 初期风格

V02 初期只采用一种美术风格，避免范围失控。

目标用户是 10 岁左右、四年级小女孩。风格关键词：

- 可爱。
- 温暖。
- 明亮。
- 圆润、可居住、可回访。
- 小镇生活。
- 宠物陪伴。
- 收集感。
- 轻 2.5D。
- Animal Crossing-like cozy town：低压力、圆润、生活模拟感强，不做绘本 / picture-book / storybook 叙事插画质感。
- Apple-like translucent glass UI：HUD、弹层和按钮容器使用轻透明、柔和模糊、清晰边界和高可读短文本；不做厚重卡片、课程面板或调试面板。

## 美术生产分类

首批素材按以下分类生产和跟踪：

- 角色 / NPC：玩家、动物居民、店主、故事角色、地图站立图、头像和表情状态。
- Sunny / 宠物：Sunny 的地图形象、小屋形象、状态表情、宠物用品关联表现。
- 小镇场景：Home、Town Plaza、Shop、Bookshop、Bus Stop、道路、草地、树、花、资源点和装饰物。
- A-Z 锚点物件：Apple Tree、Bear Corner、Clock Tower 等承载 A-Z 记忆宫殿的常驻场景物件。
- 家具与家园：可购买、可摆放、可旋转、可收起的家具和家园装饰。
- UI 与图标：HUD、底部按钮、背包、相册、商店、货币、状态、弹层和触控反馈图标。
- 音效与环境氛围：点击、购买、收集、摆放、Sunny 反馈、小镇环境音、室内环境音和地点氛围。

每个素材都要记录：

- 逻辑 asset ID。
- 用途。
- 建议尺寸或比例。
- 当前状态。
- 触控 / 观看场景。
- 替换目标。
- 验收截图要求。

## 逻辑 asset ID 规则

逻辑 asset ID 是玩法、数据和主题资源之间的稳定契约，不等同于文件名或磁盘路径。玩法脚本、内容 JSON、测试和 UI 只能引用逻辑 ID；具体 `res://...` 路径必须由 `ThemeProfile` / `AssetResolver` 统一映射。

命名规则：

- 全部使用小写 `snake_case`。
- 格式为 `<domain>.<stable_name>[.<variant>]`。
- `domain` 使用稳定分类：`character`、`pet`、`place`、`anchor`、`furniture`、`ui_icon`、`ui_skin`、`sfx`、`ambience`。
- `stable_name` 使用稳定对象 ID，不使用临时文件名、中文名、画师名、版本号或尺寸。
- `variant` 用于表现状态或用途，例如 `standing`、`portrait`、`happy`、`day`、`night`、`inventory`、`placed`、`pressed`。
- 文件可以按制作需要命名和分目录，但不能反向成为玩法合同。
- 逻辑 ID 一旦进入数据或脚本合同，除非有迁移记录，不应重命名。

推荐目录只用于 PM / 美术生产排期，不作为运行时硬编码依据：

```text
assets/art/characters/
assets/art/pets/
assets/art/places/
assets/art/anchors/
assets/art/furniture/
assets/art/ui/icons/
assets/art/ui/skins/
assets/audio/sfx/
assets/audio/ambience/
```

## 资产状态

每个素材必须标记当前状态，状态用于 PM 排期、替换优先级和验收沟通。

| 状态 | 英文值 | 使用条件 |
|---|---|---|
| 占位 | `placeholder` | 程序或临时视觉，仅用于功能验证；可以色块、简单形状或临时图，不作为最终观感验收。 |
| 升级占位 | `placeholder_plus` | 已脱离纯程序占位，能用于首屏构图、遮挡、比例和互动热点验证；仍可能需要重画或替换，不作为视觉达成依据。 |
| 草案 | `draft` | 已有明确方向、轮廓和比例，可用于截图讨论；允许风格、颜色和细节继续变化。 |
| 生产 | `production` | 可进入游戏集成和较完整体验验证；尺寸、锚点、透明边缘、状态切图和导出规范基本稳定。 |
| 已批准 | `approved` | Art Direction / PM 已确认可作为当前 V02 正式素材；只允许修 bug、压缩和适配，不随意重画。 |

旧文档里的“半正式”对应 `production`，“正式”对应 `approved`。V02.7A 审计阶段允许使用 `placeholder_plus` 区分“已脱离程序占位但还不够进入草案 / 生产”的资源。后续表格优先使用英文值，方便进入数据和工具链。

## 统一范围

同一阶段内需要统一：

- 地图。
- 角色。
- 宠物。
- NPC。
- UI。
- 图标。
- 记忆卡。
- 小游戏。
- 音频情绪。

不同时尝试像素风、3D Q 版、扁平卡通、水彩等多套方向。

## AssetResolver 约束

资产系统不能把具体路径写死在玩法逻辑里。玩法脚本只能通过逻辑 ID 请求资源。

建议设计：

```text
ThemeProfile
AssetResolver
```

运行时通过接口取资源：

```text
get_npc_sprite("mina")
get_pet_sprite("sunny")
get_tile_atlas("world")
get_card_art("a_apple_core")
get_ui_skin("primary_button")
```

V02 起新增或替换素材时必须满足：

- 玩法脚本不得新增 `load("res://...")`、`preload("res://...")` 或拼接具体美术文件路径来绑定角色、场景、家具、图标、音频。
- 内容数据可以保存逻辑 asset ID，例如 `character.mina.standing`、`furniture.wooden_chair.placed`。
- `ThemeProfile` 负责声明当前 Animal Crossing-like cozy town / Apple-like translucent glass UI 主题下每个逻辑 ID 对应的资源路径、状态和可选元数据。
- `AssetResolver` 负责按逻辑 ID 返回 Texture、AudioStream、PackedScene 或 fallback placeholder。
- 缺失映射时必须回退到明确的 placeholder，并在测试或日志里暴露，不能静默变成空白。
- A-Z anchor 的逻辑 ID 必须绑定稳定世界锚点，不得因换图破坏地图顺序、地点记忆或新词故事回访路径。

## 命名与交付

首批素材清单必须至少包含以下字段：

| 字段 | 说明 |
|---|---|
| 分类 | 角色 / NPC、Sunny / 宠物、小镇场景、A-Z 锚点物件、家具与家园、UI 与图标、音效与环境氛围之一。 |
| 对象 ID | 玩法或内容对象的稳定 ID，例如 `mina`、`sunny`、`home`、`town_plaza`、`shop`、`a.apple_tree`、`wooden_chair`。 |
| 逻辑 asset ID | 按 `<domain>.<stable_name>[.<variant>]` 生成，例如 `ui_icon.coin`。 |
| 用途 | 地图显示、头像、HUD、背包、相册、商店货架、小屋摆放、环境氛围等。 |
| 尺寸 / 比例 | 建议导出尺寸、屏幕显示比例、九宫格需求、循环音频长度或移动端可读尺寸。 |
| 状态 | `placeholder`、`draft`、`production`、`approved`。 |
| 替换目标 | 当前要替换的占位节点、现有逻辑 ID、截图区域或体验问题。 |
| 截图验收 | 至少说明验收场景、视口和判断点；UI / 地图优先使用 1280x720 横屏截图。 |

交付规则：

- 使用稳定逻辑 ID，不直接依赖文件名作为玩法约定。
- 角色、场景、家具、UI 图标和音效都要有可追踪的替换状态。
- 首批素材优先替换首屏、核心交互和高频 UI。
- 地图、角色、家具和 UI 验收时必须给出截图或录屏证据。
- 音效和环境氛围验收时必须说明触发场景、循环方式、音量层级和是否会打断温和小镇氛围。
- 英语只作为环境文字、物件名、短 NPC 句、相册标签、语音片段或可选小游戏内容；不得把素材清单组织成课程、测试、背词或 Letter Snake 主线推进。

## V02-ART-005 UI 图标与状态素材清单

本清单只覆盖 HUD、按钮和状态图标素材需求，不生成素材、不改运行时代码、不改 JSON。所有条目使用 `ui_icon.*` 逻辑 asset ID，后续进入 `ThemeProfile` / `AssetResolver` 时再映射到具体资源路径。

UI 图标定位原则：

- HUD 和按钮不再只依赖文字表达；图标必须和短文本 / 数值一起形成移动端可读入口。
- 图标服务生活 RPG / cozy town 日常：金币、点心、Sunny 状态、背包、相册、商店、设置和退出；不得做成课程、考试、打卡、家长后台或 Letter Snake 主线入口。
- 按钮图标需要支持 `normal` / `pressed` / `disabled` 或等价触控状态，触控反馈必须明显但温和，不制造误触或遮挡。
- 图标不能遮挡玩家、Sunny、NPC、A-Z anchor、资源点、商店商品、背包内容或关闭路径。
- 同一组 UI 图标需保持 Animal Crossing-like cozy town 风格，圆润边缘、明亮但不过度饱和的颜色和一致线宽；承载图标的 HUD / 弹层 / 按钮底板走 Apple-like translucent glass 方向，保证透明感、可读性和触控反馈。

优先级说明：

- `P0 高优先级`：首屏 HUD、宠物状态、背包和商店等高频入口，直接影响孩子是否知道当前有什么、能去哪里。
- `P1 核心路径`：相册、设置、退出和弹层关闭路径，支撑发布前移动端可用性。
- `P2 后续补齐`：可选状态细化、空态、通知和后续二级入口，适合在 P0 / P1 图标稳定后扩展。

P0 最小替换包是 `ui_icon.coin`、`ui_icon.sunny_snack`、`ui_icon.pet_hunger`、`ui_icon.pet_happy`、`ui_icon.bag` 和 `ui_icon.shop`。这组素材覆盖首屏资源、宠物状态、背包入口和商店入口，让高频生活操作不再只靠文字。

P1 建议紧跟 `ui_icon.album`、`ui_icon.settings`、`ui_icon.exit` 和 `ui_icon.close`。这些图标决定相册回访、设置入口、关闭路径和退出路径是否清楚，是移动端发布前体验门槛的一部分。

P2 可在 HUD 布局和主按钮风格稳定后补 `ui_icon.home`、`ui_icon.notice_dot` 以及二级状态变体。通知和状态扩展只能表达温和回访、发现和生活照顾，不得转向课程进度、打卡压力、家长后台、排名或考试入口。

UI 图标截图验收至少覆盖 1280x720 横屏：Town Plaza 首屏 HUD、Home Room 宠物状态、Shop 面板、Bag 面板、Album 面板和 Settings / Exit 弹层。验收重点是图标 + 短文本 / 数值可读、触控目标足够大、normal / pressed / disabled 状态清楚、按钮间距稳定、图标不遮挡地图角色 / Sunny / A-Z anchor / 商品内容 / 关闭路径。
