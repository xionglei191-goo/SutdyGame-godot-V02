# 首轮任务包：阶段 1 启动

> 目标：建立可运行、可测试、可继续扩展的 Godot 技术薄片。  
> 汇合验收：Godot 主场景从最小 JSON 加载并显示 Home、Town Start 和 Supermarket 占位地点。

## Task A：V02-CORE-001/002 工程骨架

**Owner：** Godot Dev Agent

**Scope：**

- 可新增 `project.godot`、`scenes/`、`scripts/`、`data/`、`assets/`、`tests/`。
- 不实现完整地图编辑器、真实语音、AI 或最终美术。

**Deliverables：**

- Godot 4.x 横屏项目，逻辑分辨率 1280x720。
- 可运行的 `Main` 场景和基础 UI shell。
- 约定目录与最小 README/运行说明。

**Acceptance Criteria：**

- `godot --headless --path . --quit` 正常退出。
- `godot --path .` 可进入主场景。
- 无硬编码真实资产路径，无真实联网或录音权限。

## Task B：V02-MAP-001 最小数据合同

**Owner：** Map Tool Agent

**Scope：**

- 可新增 `scripts/data/`、`data/maps/` 和对应 `tests/`。
- 只定义技术薄片需要的 World、District、Road、Place 和 MemoryAnchor 字段。

**Deliverables：**

- 最小 Resource 类型和 `world_map.json` 示例。
- 示例包含 Home、Town Start、Supermarket，以及 A/B/C/D/K/O/S/T/W 九个 anchor。
- 数据字段说明和首批校验规则。

**Acceptance Criteria：**

- JSON 可被 Godot 加载。
- 所有对象 ID 唯一。
- place 位于合法 district，interaction cell 不与 occupied cell 重叠。
- anchor `route_order` 唯一，核心编码与 A-Z 基线一致。

## Task C：开工决策确认

**Owner：** PM Agent / 主 Agent

**Deliverables：**

- 确认首批启用 9 个 anchor。
- 确认首个宠物使用 Sunny/Dog 占位。
- 确认 Android 首测设备或最低屏幕基线。
- 确认自动测试 runner。

**Acceptance Criteria：**

- 决策记录写入 `docs/15_项目经理接管与下一阶段执行计划.md` 或独立决策日志。
- 未确认项不得阻塞工程骨架；使用文档中的建议值作为临时默认值。

## 汇合任务：V02-RUNTIME-001 最小加载

Task A 和 Task B 通过后，由 Godot Dev Agent 实现最小 `RuntimeMapBuilder`：

- 从 `world_map.json` 加载数据。
- 显示地面、道路和三个地点占位。
- 输出明确的数据加载或校验错误。
- 提供 headless smoke check。

完成后由 QA Agent 按数据合同、运行结果和孩子端可见内容进行首轮验收。
