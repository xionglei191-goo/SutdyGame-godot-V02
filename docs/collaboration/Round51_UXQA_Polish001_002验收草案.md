# Round 51 UX/QA V02-POLISH-001/002 验收草案

> 日期：2026-06-05  
> 小组：UX / QA  
> 范围：`V02-POLISH-001 退出与设置入口`、`V02-POLISH-002 玩家路径截图验收` 的文档验收草案。  
> 本草案只定义发布前质量门槛，不修改运行时代码、data、tests、todo 或 lessons。

## 目标

本轮把 Sunshine Town 的“能玩”推进到“可验收”：孩子端必须通过真实可见入口完成核心路径，PC / 桌面调试必须有可见退出路径，移动端不得把退出或设置放进主操作误触区。截图验收必须保护生活 RPG / cozy town 主体验，不得用隐藏 contract 按钮、调试面板或服务调用截图替代玩家路径。

参考经验：

- `LESSON-007`：主循环验收必须包含首屏视觉和玩家感知，不能只验收服务、节点和数据加载。
- `LESSON-009`：隐藏 contract 按钮不能证明孩子端可玩，必须从孩子端真实可见按钮和面板完成操作。

## V02-POLISH-001 退出与设置入口验收草案

### PC / 桌面退出入口

PC / 桌面调试环境需要一个可见但低干扰的退出路径：

| 项目 | 验收要求 | 失败判定 |
|---|---|---|
| 入口位置 | 退出入口放在设置 / 安全面板内，或桌面调试专用角落入口；不放在底部主操作栏。 | 退出按钮与 `看看`、`小镇`、`小屋`、`背包` 等主操作并列，容易误触。 |
| 可见性 | PC 视口下可通过孩子端可见 UI 打开设置 / 安全面板，再看到退出或返回安全状态选项。 | 只能通过键盘、隐藏按钮、脚本调用、contract 按钮或调试命令退出。 |
| 确认方式 | 退出前需要二次确认，文案温和，例如“要先休息一下吗？”、“回到安全位置”。 | 单击立即退出、使用恐吓或损失提示，或暗示今天必须完成内容。 |
| 工程信息 | 孩子端不显示 Godot、debug、runner、JSON、contract、test、script 等工程词。 | 退出 / 设置面板出现工程状态、测试入口、数据加载日志或合同按钮。 |
| 安全状态 | 若不直接关闭游戏，也可以回到 Home / 小镇安全位置或暂停状态。 | 退出后进入空白、报错、未设计调试页，或进入家长后台 / 工程面板。 |

建议 PC 文案：

- 设置入口：`设置`
- 休息确认：`要先休息一下吗？`
- 取消：`继续逛小镇`
- 确认：`回到安全位置` 或 `退出游戏`

禁用文案：

- `Quit Debug`、`Close Runner`、`Contract Test`、`JSON Loaded`
- “今天还没完成”“连续回来才有奖励”“家长会看到”
- “失败”“错过”“损失”“扣除”

### 儿童端设置 / 安全入口

儿童端设置入口应是低频、安全、可理解的入口，不抢主循环：

| 项目 | 验收要求 | 失败判定 |
|---|---|---|
| 入口层级 | 设置 / 安全入口放在顶部角落、暂停面板或安全面板中，不进入底部高频主操作栏。 | 设置、退出、家长、测试、相册、商店和小屋全部堆进底栏，挤压主操作。 |
| 图标与文字 | 可使用齿轮或小屋安全图标，配短文本 `设置`；移动端触控区域清楚。 | 只用缩写、工程词或过小图标，孩子无法判断含义。 |
| 内容范围 | 首批只放声音、返回安全位置、退出 / 休息确认等低风险项。 | 加入真实账号、联网、录音、云服务、家长报告或开放社交入口。 |
| 返回路径 | 设置面板必须有明显关闭 / 返回按钮，回到打开前状态或 Home。 | 进入设置后无法返回，或关闭按钮被 HUD / 底栏遮挡。 |
| 主体验保护 | 设置入口不遮挡角色、Sunny、资源点、NPC、A-Z anchor 和主要移动区域。 | 设置面板默认覆盖大部分地图，启动后首先看到设置而非 Sunshine Town。 |

### 移动端不误触规则

移动端视口下，退出与设置必须遵守以下规则：

- 退出按钮不得常驻底部主操作栏。
- 退出按钮不得靠近移动摇杆、`看看`、`小镇`、`小屋`、`背包`、商店购买、家具摆放 / 旋转 / 收起等高频按钮。
- 设置入口需要二级确认才能退出；第一次点击只打开设置或安全面板。
- 关闭 / 返回按钮应始终可见，不能被刘海 / 安全区、顶部 HUD、底栏或弹层边缘遮挡。
- 横屏较小视口下，如果空间不足，优先隐藏退出深层入口，不压缩主操作按钮到误触尺寸。
- 不用红色大按钮、倒计时、警告语或损失提示制造压力。

## V02-POLISH-002 玩家路径截图验收草案

截图验收以 1280x720 横屏为基础视口。后续 `V02-POLISH-003` 复核较小横屏视口，`V02-POLISH-004` 复用同一截图点检查正式 / 半正式素材替换。

### 关键路径 1280x720 截图清单

| 编号 | 截图点 | 入口路径 | 操作 / 玩家位置 | 必须可见 | 不得出现 |

| S01 | 小镇首屏 | 启动主场景 | 玩家在 Home / Town Plaza 初始可见区域 | Sunshine Town 生活小镇身份、玩家、Sunny 或主要 NPC、Home / Plaza / 道路、顶部 HUD、底部主操作栏 | skeleton、Loaded from JSON、debug、contract、测试按钮、裸工程面板 |

| S03 | 资源收集 | 小镇可见资源点 | 玩家靠近树枝 / 小花 / 石子并触发收集 | 资源点、收集反馈、金币 / 背包变化提示、主操作仍可读 | 隐藏按钮触发证据、资源像 UI 图标贴在地图上、反馈遮挡底栏 |
| S04 | 背包气泡 | 点击底部 `背包` | 背包气泡打开 | 背包内容、相册入口或物品入口、关闭路径、底栏不被遮挡到无法操作 | 空白面板、调试 ID 列表、文字溢出、隐藏 contract 按钮 |
| S05 | 商店货架 | 靠近 Shop 或点击可见 Shop 入口 | 商店货架面板打开 | 店长 / Shop 上下文、商品、价格、金币、关闭路径 | 买不起羞辱、强制购买、工程商品 ID 裸露为主文案 |

### 截图触发规则

以下改动后必须补截图验收：

- 主场景、HUD、底部栏、设置 / 安全面板、背包、商店、小屋、相册、NPC 对话或 anchor 互动 UI 改动。
- 任意 P0 / P1 场景、角色、Sunny、家具、UI 图标或 A-Z anchor 素材替换。
- 删除、移动、合并或重命名孩子端可见入口。
- 新增每日轻委托、商店商品、小屋家具操作或相册入口。
- 为移动端做布局压缩、按钮调整、安全区调整或弹层改版。

## 统一失败判定标准

任一截图或验收路径出现以下问题，应判定失败：

- 使用隐藏 contract 按钮、调试面板、服务调用、脚本直调或测试入口作为孩子端可玩证据。
- 首屏或弹层出现 Godot、debug、runner、JSON、contract、test、script、skeleton、Loaded from JSON 等工程文案。
- 退出、设置、购买、任务或宠物照顾文案出现失败惩罚、连续登录压力、学习测试、家长报告、陌生人社交暗示或亏欠感。
- 退出按钮位于底部主操作误触区，或单击立即退出且没有温和确认。
- 关键路径没有明确关闭 / 返回按钮，或关闭按钮被 HUD、底栏、安全区、弹层边缘遮挡。
- 文字溢出按钮 / 面板，覆盖关键地图物件、玩家、Sunny、NPC、资源点或 A-Z anchor。
- 商店、相册、小屋、家具操作等功能只能通过内部方法完成，孩子端没有真实可见入口。
- A-Z anchor 被表现成裸字母测验牌、课程入口、背诵检查或 Letter Snake 主线门槛。
- 1280x720 截图中首屏仍像程序化占位原型，无法感知 Sunshine Town 生活小镇主体验。
