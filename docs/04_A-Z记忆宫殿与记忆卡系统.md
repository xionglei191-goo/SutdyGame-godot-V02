# A-Z 记忆宫殿与记忆卡系统

## A-Z 的定位

A-Z 是 V02 的核心记忆宫殿和世界地图底层编码。它不应以考试、背诵或课程表形式压到孩子面前，但必须在地图、地标、物件、NPC 台词、收藏和路线中稳定存在。

上层玩法以生活 RPG / 小镇养成为主。孩子可以自由探索、装饰家园、和动物居民互动、收集资源、照顾伙伴。A-Z 核心编码保持稳定，并通过每个字母对应的世界锚点进行潜意识植入。

V02 复用以下文档作为 A-Z 记忆宫殿核心基线：

```text
docs/assets/Sunshine_Town_And_The_World_A-Z字母记忆宫殿策划_v0.1.md
```

该文档中的 A-Z 主编码、固定位置、路线回忆和视觉权重原则，是 V02 世界地图、地点布局、收藏图鉴、小游戏、语音和 NPC 台词的基础。

## 地图植入原则

- 26 个字母都必须有稳定 `anchor_id`、核心词、地图位置和可识别场景表现。
- 锚点不是悬浮 UI 图标，而应是生活世界中的地标、家具、道具、店铺、自然物或 NPC 事件。
- 首批可只开放 A/B/C/D/K/O/S/T/W 的互动，但其他字母的位置和路线顺序必须预留。
- 玩家不需要被告知“正在学习记忆宫殿”；系统通过反复路过、对话、收集和装饰让编码生根。
- 任何主题包或地图改版不得改变核心字母编码和路线顺序，只能改变表现皮肤。

## 三层结构

1. Core Anchor：固定 A-Z 编码，例如 A = Apple。
2. World Placement：每个 anchor 在世界中的固定位置、物件和场景表现。
3. Subconscious Usage：探索、装饰、NPC、小游戏、语音和故事如何自然调用 anchor。

核心编码不应被普通内容随意覆盖。扩展词可以开放，例如 A 可以有 Apple core card，也可以有 Ant、Airplane 等扩展卡，但核心锚点保持稳定。

## 新单词故事规则

后续每个新单词都必须围绕记忆宫殿编码来讲解，不能孤立加入词表。新增单词时必须定义：

- `word`：单词本体。
- `letter`：归属字母。
- `core_anchor_id`：挂靠的 A-Z 核心锚点。
- `world_place_id`：发生在哪个地图地点。
- `story_memory`：孩子能想象的一句话故事画面。
- `visual_hook`：画面中最醒目的记忆钩子。
- `npc_or_event_ref`：由哪个 NPC、事件、家具、资源或小镇变化触发。
- `review_path`：玩家如何在生活中再次遇到它，例如路过、摆放、送礼、商店购买或图鉴回看。

示例：

```text
word: ant
letter: A
core_anchor_id: anchor_a_apple
world_place_id: town_plaza
story_memory: 一只小 ant 把 apple tree 下的红苹果当成小山搬家。
visual_hook: apple tree 根部的蚂蚁路线
npc_or_event_ref: Mina asks you to look near the apple tree.
review_path: 每次路过 Apple tree 都能看到蚂蚁路线和 apple 锚点。
```

这条规则适用于 Memory Card、NPC 台词、资源、家具、任务、小游戏目标和后续内容包。

## 记忆卡系统

每个单词可以设计成一张记忆卡。记忆卡绑定：

- 字母。
- 单词。
- 核心锚点。
- 图片。
- 语音。
- 场景。
- 故事记忆。
- 视觉钩子。
- 回访路径。
- 小游戏记录。
- NPC 记忆。
- 收集状态。

孩子不是在背单词，而是在收集记忆卡和保存故事画面。

## 记忆相册

Memory Album 是孩子的记忆收藏空间。

例如：

```text
A - Apple
画面：Welcome Box 里的红苹果
故事：Mina helped you open the box.
声音：apple
地点：Home
小游戏：Letter Snake
状态：Collected / Shiny
```

## 小游戏与卡片

小游戏是卡片采集方式之一。

例如字母贪吃蛇：

- 吃到正确字母和单词。
- 分数越高，卡片点亮越多。
- 低分也有小奖励。
- 高分获得更多金币、闪卡或特殊画面。

不使用失败压力，只使用奖励差异。
