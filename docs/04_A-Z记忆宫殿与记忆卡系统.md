# A-Z 记忆宫殿与记忆卡系统

## A-Z 的定位

A-Z 是 V02 的固定认知编码底座。

上层玩法可以开放变化，但 A-Z 核心编码必须稳定。孩子可以自由探索小镇、宠物、商店、小游戏和 NPC，但底层记忆编码从一开始就植入世界。

V02 复用以下文档作为 A-Z 记忆宫殿核心基线：

```text
docs/assets/Sunshine_Town_And_The_World_A-Z字母记忆宫殿策划_v0.1.md
```

该文档中的 A-Z 主编码、固定位置、路线回忆和视觉权重原则，是 V02 后续地图、记忆卡、小游戏、语音和 AI NPC 调用的基础。

## 三层结构

1. Core Anchor：固定 A-Z 编码，例如 A = Apple。
2. World Placement：每个 anchor 在世界中的位置、物件和场景表现。
3. Gameplay Usage：任务、小游戏、语音、AI NPC、复现事件如何调用 anchor。

核心编码不应被普通内容随意覆盖。扩展词可以开放，例如 A 可以有 Apple core card，也可以有 Ant、Airplane 等扩展卡，但核心锚点保持稳定。

## 记忆卡系统

每个单词可以设计成一张记忆卡。记忆卡绑定：

- 字母。
- 单词。
- 图片。
- 语音。
- 场景。
- 故事记忆。
- 小游戏记录。
- NPC 记忆。
- 收集状态。

孩子不是在背单词，而是在收集记忆卡和保存故事画面。

## 记忆相册

Memory Album 是孩子的记忆收藏空间。

例如：

```text
A - Apple
画面：Welcome Box 里的红苹果
故事：Mina helped you open the box.
声音：apple
地点：Home
小游戏：Letter Snake
状态：Collected / Shiny
```

## 小游戏与卡片

小游戏是卡片采集方式之一。

例如字母贪吃蛇：

- 吃到正确字母和单词。
- 分数越高，卡片点亮越多。
- 低分也有小奖励。
- 高分获得更多金币、闪卡或特殊画面。

不使用失败压力，只使用奖励差异。
