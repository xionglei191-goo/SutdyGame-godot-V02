# V02 开发路线

## 阶段原则

V02 不应该一开始就堆功能。先建立正确的产品框架、数据结构、编辑工具和运行时主链。

后续开发继续采用子 agent 分工协作模式，协作规范沿用并升级到 `docs/collaboration/多Agent协作规范_v0.2.md`，具体任务交接使用 `docs/collaboration/任务交接模板.md`。

可执行开发计划以 `docs/13_V02详细开发计划.md` 为准。该计划定义第一阶段任务级清单、核心数据结构、系统接口、文档要求、测试验收和远景开发方向。

## 阶段 0：框架确认

产出：

- V02 文档基线。
- 产品总纲。
- 学习阶梯。
- A-Z 记忆宫殿规则。
- 内容基线与首批内容规划。
- 地图编辑架构。
- 技术架构。

## 阶段 1：Godot 项目骨架

产出：

- 新 Godot 项目。
- 基础目录结构。
- ThemeProfile 和 AssetResolver。
- Local SaveService。
- 基础 UI shell。

## 阶段 2：地图编辑与运行时生成

产出：

- WorldOverviewScene。
- EditorOnly proxy nodes。
- Grid overlay。
- Place marker 拖动。
- Road cell 编辑。
- Occupied / interaction cell 编辑。
- 写回数据。
- 地图校验。

## 阶段 3：A-Z 与 Memory Card

产出：

- MemoryCard 数据。
- Memory Album UI。
- 卡片收集状态。

## 阶段 4：Home/Pet/Town 主循环

产出：

- Home。
- Pet。
- Shop。
- Coins。
- 简单任务。

## 阶段 5：小游戏框架

产出：

- Letter Snake 原型。

## 阶段 6：语音与 NPC

产出：

- TTS 接口。
- 录音接口。
- 固定 NPC dialogue。
- NPC profile/memory 文件。
- AI LLMClient 预留。

## 阶段 7：家长后台与本地摘要

产出：

- 设置页或家长 NPC 入口。
- 本地 dashboard。

- AI 摘要接口预留。
