## 当前完成事实

- 可探索小镇：玩家移动、触屏控制、Home / Town Plaza / Shop 等小镇区域、NPC、资源、商店和家园装饰已接通。
- 每日系统：每日问候、每日资源刷新、轻委托、今日状态和本地保存已通过 smoke。
- 我的小屋：独立小屋视图、家具摆放/旋转/收起、宠物用品、Sunny 家内反馈和保存重载已通过。
- A-Z 场景化：26 个 A-Z anchor 常驻世界结构，首批 A/B/C/D/K/O/S/T/W 已强化为小镇物件、相册收藏和新词回访路径。
- 内容合同：每日委托、NPC 对话、家具商品、anchor 内容包和新词故事都有数据合同与校验。
- 真实可玩入口：相册、商店购买、小屋家具操作等核心动作已有孩子端可见入口，并通过玩家操作级 smoke。
- 美术接入链路：`ThemeProfile` / `AssetResolver` 已能按逻辑 asset ID 接入 PNG/SVG 资源；V02.7A 已用 1280x720 与 960x540 截图建立当前 production 基线，但后续仍不得把缺少新截图证据的大改动直接标成 `approved`。
- V02.8 纵切：孩子端已能通过真实可见入口完成 Mina 委托、资源采集、奖励、商店购买、小屋摆放、Sunny 反馈和 C/O/S 回访，并注册进 `headless_runner`。
- V02.9 回访节奏：`local_day_001` 至 `local_day_007` 的今日状态、商店轮换、P0 可见路径、P1 不阻断边界和 1280x720 / 960x540 代表截图已通过。
- V02.10 P1 居民回访：Bookshop / Bear Corner / Bus Stop / Taxi marker 已有真实可见入口、两条 P1 轻回访、B/T 相册记录、P1 smoke 和双视口截图证据。

历史的 Memory Card、Letter Snake、Voice / AI / Parent stub 仍作为技术资产和可选系统保留。它们不再驱动主循环。

## 下一阶段目标

## 四条并行路线

### 工程路线

- 维护现有可玩闭环，不把已完成系统改回调试面板或隐藏 contract 按钮。
- 所有“可玩功能”必须有孩子端可见入口，且至少一个测试通过可见按钮或面板按钮触发。
- 玩法脚本不得硬编码具体素材路径，只能通过逻辑 asset ID 和 `AssetResolver` 请求资源。
- 新增数据内容优先改 JSON / 数据合同 / focused test，避免把内容规则重新塞回 `scripts/main.gd`。

### 策划路线

- NPC 问候、关系阶段、轻委托、商店轮换、节日天气和 A-Z 场景故事要形成可复用生产规范。
- 每条每日委托必须有 NPC、目标、奖励、关系反馈、重复规则、儿童安全文案和可见操作入口。
- 经济设计保持非惩罚：买不起只给温和反馈，不制造打卡压力、损失惩罚或强消耗循环。
- 英语继续作为环境层、收藏层、NPC 短句和记忆宫殿故事层，不作为考试、课程表或主线门槛。

### 美术路线

- 初期只采用单一 storybook / cozy town 风格，不并行扩展多套主题。
- 所有素材需要用途、逻辑 asset ID、尺寸/比例、状态、替换优先级和验收截图。
- 首批素材范围覆盖角色/NPC、Sunny/宠物、小镇场景、A-Z 锚点物件、家具与家园、UI 与图标、音效与环境氛围。
- UI 和 HUD 逐步从纯文字表达转向图标+短文本表达，但必须保证移动端可读和可触控。
- `production` 只表示资源可进入游戏集成，不等同于美术最终质量；`approved` 必须有 PM / Art Direction 确认和 1280x720、960x540 截图证据。
- V02.7A 前不得继续把缺少截图证据的生成图或程序占位图当作产品视觉完成。

### QA 路线

- 数据合同继续拦截重复 ID、非法价格、儿童安全文本、缺失记忆宫殿字段和覆盖核心 A-Z anchor。
- 玩家路径 smoke 继续覆盖小镇、背包、商店、小屋、相册、anchor、NPC、资源和家具操作。
- 截图验收覆盖 1280x720 和较小横屏视口，检查遮挡、文字溢出、安全区、按钮尺寸和关闭路径。
- 发布前质量门槛必须包含退出/设置入口、移动端触控复核和首批正式或半正式素材替换。
- V02.8 纵切验收必须覆盖从启动到完成一段日常生活的玩家路径，不能只验证服务或隐藏 contract 方法。
- V02.9 已完成 7 天回访节奏；后续改动必须继续保持无连续登录奖励、错过惩罚或限时压力。
- V02.10 只允许把 Bookshop / Bus Stop 做成 P1 支线回访；未完成真实入口、截图和儿童安全验收前，不得把它们写入 P0 主流程硬依赖。
- V02.11 只做天气 / 轻事件的生活化纵切，不做真实天气联网、运营限时活动、连续登录、补签、排行榜或节日奖励墙。

## 持续门槛

- 每个可玩功能必须有玩家可见入口，隐藏 contract 按钮不能作为完成依据。
- 每个素材必须有稳定逻辑 asset ID，不能由玩法脚本硬编码具体路径。
- 每个新词必须绑定 `letter`、`core_anchor_id`、`world_place_id`、`story_memory`、`visual_hook` 和 `review_path`。
- 每次 UI 或美术大改必须补截图验收，不能只跑 headless。
- 儿童端不得出现失败惩罚、课程压力、家长报告口吻、开放陌生人社交或工程调试文案。
