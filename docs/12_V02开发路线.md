# V02 开发路线

## 阶段原则

V02 先做正确的生活 RPG / 小镇养成骨架，同时把 A-Z 记忆宫殿作为世界地图底层编码放进去。英语内容自然出现，记忆宫殿通过地图锚点、路线、地标和收藏进行潜意识植入。

开发顺序以“小镇生活 MVP”为核心：移动、探索、A-Z 地图锚点、NPC、收集、商店、家园装饰、存档。Letter Snake 和家长后台在主循环稳定后再作为可选系统接入。

## 阶段 0：方向纠偏与基线

产出：

- 产品定位改为生活 RPG / 小镇养成。
- 明确英语自然融入，不驱动主循环。
- 明确 A-Z 记忆宫殿是地图底层编码，不是可删除的装饰系统。
- 重写 `todo.md`、`AGENTS.md` 和核心玩法循环。
- 标记旧学习闭环为技术资产。

## 阶段 1：角色与小镇探索

产出：

- 可移动玩家角色。
- Home、Town Plaza、Shop 三个可走区域。
- 首批 A-Z 锚点在地图中有固定位置和可感知表现。
- 基础相机、碰撞和触屏移动。
- 从 JSON 生成可探索场景。

## 阶段 2：NPC 关系与轻委托

产出：

- 2-3 个动物居民。
- 固定对白、每日问候、好感。
- 轻委托数据：收集、购买、送礼、拜访。
- 对话和关系存档。

## 阶段 3：资源、背包与商店

产出：

- 地图资源点。
- 背包与材料。
- Shop 商品表。
- coins、购买、售卖或兑换。

## 阶段 4：家园装饰 MVP

产出：

- 家具数据。
- 房间网格。
- 摆放、移动、收起家具。
- 房间状态保存与加载。

## 阶段 5：宠物与陪伴

产出：

- 宠物状态。
- 喂食、抚摸、跟随或房间互动。
- 宠物心情反馈。

## 阶段 6：收藏与英语环境

产出：

- Collection Album。
- A-Z 彩蛋和物品图鉴。
- 简单英文标签和语音。
- Letter Snake 作为可选小游戏。

## 阶段 7：家长后台与本地摘要

产出：

- 家长入口。
- 本地生活摘要：游玩时间、收集、NPC 互动、装饰进展。
- 不进入孩子主流程。

## 阶段 8：远期预留

暂不实现，但架构预留：

- 账号。
- 云存档。
- 内容包更新。
- 受控互访。
- 节日活动。
- AI NPC 实时开放对话。
