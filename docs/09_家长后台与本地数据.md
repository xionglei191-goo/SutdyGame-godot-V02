# 家长后台与本地数据

## 总原则

家长后台不进入孩子主游戏流程。孩子端只看到奖励、宠物、小镇变化、卡片、NPC 和故事。家长端通过设置页或特定 NPC 激活后进入后台。

## V02 最小家长入口

家长入口不在孩子主导航中显示。V02 最小规格：

- 普通孩子端主流程不显示 `Parent`、`Admin`、`Dashboard`、`Report`、`后台`、`家长`、`报告`。
- 预留入口为设置页或角落的隐藏长按，不由 NPC、地图热点或生活任务自然触发。
- 长按建议 3 秒，并通过本地成人确认 stub 后再进入。
- 不需要账号、网络、录音或真实 AI。
- 进入后切换到独立家长层或场景；退出后返回设置页或标题页，不自动改变孩子端 Home/Map 状态。

当前代码规格由 `Main.get_parent_entry_spec()` 暴露，完整家长摘要界面归 `V02-PARENT-004`。

## 家长可见内容

- 游戏时间。
- 接触过的词汇和句型。
- 记忆卡收集状态。
- 小游戏表现摘要。
- 语音练习记录。
- AI NPC 对话摘要。
- 宠物和任务进展。
- 建议复现内容。

## 本地优先

V02 初期是本地游戏，不做真实账号和联网后台。

但数据结构预留：

- `player_id`
- `profile_id`
- `save_slot`
- `learning_record_id`
- `account_adapter`

未来可以替换成本地/云端实现：

```text
SaveServiceLocal -> SaveServiceCloud
ParentDashboardLocal -> ParentDashboardBackend
NPCMemoryLocal -> NPCMemoryCloud
```

## 隐私原则

即使暂时自用，也需要从架构上尊重儿童数据边界：

- 录音可开关。
- AI 可开关。
- 联网可开关。
- 家长可查看摘要。
- 未来支持删除数据。
