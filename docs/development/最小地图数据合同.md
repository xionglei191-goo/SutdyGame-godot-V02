# 最小地图数据合同

> 对应任务：V02-MAP-001  
> 运行时事实来源：`data/maps/world_map.json`  
> 编辑器数据模型：`scripts/data/*_resource.gd`

## 目标

本合同定义第一阶段技术薄片需要的 World、District、Road、Place 和 MemoryAnchor 最小字段。Godot 运行时应从 JSON 读取地图数据；Resource 类型用于后续 Godot 内编辑器、Inspector 配置和导出流程。

## 顶层字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `map_id` | String | 世界地图稳定 ID，全局唯一。 |
| `version` | Number | 数据版本，首版为 `1`。 |
| `canvas_size` | Object | 地图格子尺寸，字段为 `w`、`h`。 |
| `cell_size` | Object | 单格像素尺寸，字段为 `w`、`h`。 |
| `theme_id` | String | 当前占位主题 ID。 |
| `districts` | Array | 区域列表。 |
| `roads` | Array | 道路层列表。 |
| `places` | Array | 地点列表。 |
| `memory_anchors` | Array | A-Z 记忆锚点列表。 |
| `collision_cells` | Array | 运行时碰撞格子。 |
| `interaction_cells` | Array | 运行时交互格子。 |

## District

| 字段 | 类型 | 说明 |
|---|---|---|
| `district_id` | String | 区域稳定 ID，必须全局唯一。 |
| `label` | String | 编辑器和调试显示名。 |
| `rect_cells` | Object | 区域矩形，字段为 `x`、`y`、`w`、`h`。 |
| `tile_id` | String | 运行时地面占位 tile ID。 |
| `allowed_place_types` | Array[String] | 该区域允许放置的地点类型。 |

首版包含 `district_home`、`district_town`、`district_school`、`district_transport`。

## Road

| 字段 | 类型 | 说明 |
|---|---|---|
| `road_id` | String | 道路稳定 ID，必须全局唯一。 |
| `label` | String | 编辑器和调试显示名。 |
| `road_type` | String | 道路类型，例如 `main`、`shop`、`transport`。 |
| `cells` | Array[Cell] | 道路经过的格子。 |
| `connects_to` | Array[String] | 相连道路 ID。 |

首版道路包括 Home 到 Town、Shop Street、Transport Link。Shop 和 Transport 交互点必须落在道路格或与道路格相邻。

## Place

| 字段 | 类型 | 说明 |
|---|---|---|
| `place_id` | String | 地点稳定 ID，必须全局唯一。 |
| `label` | String | 孩子端或调试显示名。 |
| `place_type` | String | 地点类型，必须在所属 district 的 `allowed_place_types` 内。 |
| `district_id` | String | 所属区域 ID。 |
| `position` | Cell | 地标主坐标，必须位于所属 district 内。 |
| `size` | Object | 地标占用尺寸，字段为 `w`、`h`。 |
| `landmark_asset_id` | String | 占位美术资源 ID，不指向真实资产路径。 |
| `occupied_cells` | Array[Cell] | 地标碰撞格子。 |
| `interaction_cell` | Cell | 入口或点击交互格，不能与 `occupied_cells` 重叠。 |
| `place_action` | String | 运行时触发动作 ID。 |
| `card_actions` | Array[String] | 该地点关联的首批卡片或锚点卡片 ID。 |
| `unlock_rule` | String | 解锁规则 ID，默认 `default_unlocked`。 |

首版数据只包含 `Home`、`Town Start`、`Supermarket` 三个最小地点。T/W 交通锚点暂绑定到 `Town Start`，坐标落在 Transport Link 方向的可见道路侧，后续可在不改变 A-Z 编码的前提下迁移到正式交通地点。

## MemoryAnchor

| 字段 | 类型 | 说明 |
|---|---|---|
| `anchor_id` | String | 锚点稳定 ID，必须全局唯一。 |
| `letter` | String | A-Z 单字母。 |
| `core_word` | String | 固定主编码词。 |
| `route_order` | Number | 完整 A-Z 固定路线顺序，不按首批启用列表重新编号。 |
| `place_id` | String | 绑定地点 ID。 |
| `position` | Cell | 地图锚点坐标。 |
| `card_id` | String | 绑定 Memory Card ID。 |
| `audio_id` | String | 绑定语音 ID，占位即可。 |
| `unlock_rule` | String | 解锁规则 ID。 |

地图数据必须常驻 26 个 A-Z anchor。第一阶段重点互动/默认解锁 9 个 anchor：

| Letter | Core Word | `route_order` |
|---|---|---|
| A | Apple | 1 |
| B | Bear | 2 |
| C | Clock | 3 |
| D | Dog | 4 |
| K | Kite | 11 |
| O | Orange | 15 |
| S | Sun | 19 |
| T | Taxi | 20 |
| W | Watch | 23 |

## 校验规则

自动或手工校验至少覆盖：

1. 顶层必填字段存在。
2. 所有 `district_id`、`road_id`、`place_id`、`anchor_id`、`interaction_id` 全局唯一。
3. `place.district_id` 必须引用已存在 district。
4. `place.place_type` 必须包含在所属 district 的 `allowed_place_types` 中。
5. `place.position` 必须位于所属 district 的 `rect_cells` 内。
6. `place.interaction_cell` 不得与该 place 的 `occupied_cells` 重叠。
7. 顶层 `interaction_cells[*].cell` 不得与顶层 `collision_cells` 重叠。
8. `memory_anchors[*].place_id` 必须引用已存在 place。
9. `memory_anchors[*].route_order` 必须唯一。
10. `memory_anchors` 必须包含完整 26 个 A-Z 世界锚点；首批未互动字母也要保留地图位置和路线顺序。
11. `route_order` 必须使用完整 A-Z 固定顺序：A=1，B=2，...，Z=26；首批 9 个 anchor 不重新压缩编号。
12. A-Z 核心编码不得被覆盖；首批重点互动固定为 A Apple、B Bear、C Clock、D Dog、K Kite、O Orange、S Sun、T Taxi、W Watch。
13. Shop 和 Transport 类地点必须通过 `interaction_cell` 连接道路格或相邻道路格。

`scripts/data/world_map_contract.gd` 提供可复用的最小字典校验入口：`WorldMapContract.validate_map(map_data)`。
