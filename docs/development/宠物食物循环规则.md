# 宠物食物循环规则

> 任务：V02-LOOP-001  
> 数据入口：`data/quests/pet_food_loop.json`  
> 范围：只定义第一阶段宠物饥饿、小游戏奖励、购买食物、喂 Sunny 的状态与奖励合同。

## 事件链

第一版主循环固定为：

```text
Welcome Home -> Meet Sunny -> Snack Time -> Food Trip -> Letter Snake -> Food Basket -> Feed Sunny
```

对应事件 ID：

| 顺序 | Event ID | 孩子端标题 | 地点 | 主要作用 |
|---|---|---|---|---|
| 1 | `event_welcome_home` | Welcome Home | `place_home` | 到家，建立起点状态 |
| 2 | `event_meet_sunny` | Meet Sunny | `place_home` | 认识 Sunny，开启宠物状态 |
| 3 | `event_snack_time` | Snack Time | `place_home` | Sunny 变饿，引出食物需求 |
| 4 | `event_food_trip` | Food Trip | `place_town_start` | 出门去小镇 |
| 5 | `event_letter_snake_food` | Letter Snake | `place_town_start` | 获得 coins 和卡片进度 |
| 6 | `event_food_basket` | Food Basket | `place_supermarket` | 购买 Sunny Snack |
| 7 | `event_feed_sunny` | Feed Sunny | `place_home` | 喂食，更新宠物、卡片和本地保存 |

## 状态输入

后续 `PetService`、`ShopService`、`QuestEventService` 只需要依赖这些输入：

| 字段 | 用途 |
|---|---|
| `current_event_id` | 当前事件 |
| `current_place_id` | 当前地点 |
| `coins` | 当前 coin 数 |
| `inventory.food_pet_snack` | Sunny Snack 持有数量 |
| `pet.hunger` | Sunny 饥饿值，0-100，越高越饿 |
| `pet.happy` | Sunny 开心值，0-100 |
| `pet.fed_today` | 本轮是否已喂食 |
| `cards` | Memory Card 状态 |
| `flags` | 主循环进度标记 |
| `minigame.letter_snake_food_run` | Letter Snake 本轮结果 |

## 状态输出

每个事件最多输出这些状态：

| 字段 | 规则 |
|---|---|
| `next_event_id` | 下一个事件；`Feed Sunny` 后为空 |
| `current_place_id` | 事件发生后的地点 |
| `coins` | 可增加或扣除，但不得低于 0 |
| `inventory.food_pet_snack` | 购买加 1，喂食减 1 |
| `pet.hunger` | `Snack Time` 设为 70；喂食后减少 25，最低 0 |
| `pet.happy` | `Meet Sunny` 设为 45；`Snack Time` 设为 35；喂食后增加 20，最高 100 |
| `pet.fed_today` | `Feed Sunny` 后为 true |
| `cards` | 只记录 seen / played / collected / shiny 等内部收集状态 |
| `flags` | 记录事件进度和闭环完成 |
| `save_snapshot` | `Feed Sunny` 后写入本地保存 |

## Coins 与奖励

Letter Snake 的 Food Set 使用 A/O/E：

| 结果带 | 最低分 | Coins | Card spark | 说明 |
|---|---:|---:|---:|---|
| `try` | 0 | 3 | 1 | 任意结果都有小奖励 |
| `bright` | 4 | 6 | 2 | 可购买 1 个 Sunny Snack |
| `shiny` | 8 | 10 | 3 | 可直接推进 1 张卡片收集 |

非惩罚规则：

- `coin_floor_per_letter_snake_run = 3`，一次游玩最低给 3 coins。
- 低表现只给较少奖励，不扣 coins、不降低卡片进度、不回退事件。
- Food Basket 需要 6 coins；如果 coins 不足，事件服务应引导再次玩 Letter Snake，而不是显示挫败反馈。

## 食物购买

`Food Basket` 使用 `reward_rules.buy_pet_food`：

| 项 | 值 |
|---|---|
| 食物 ID | `food_pet_snack` |
| 孩子端名称 | Sunny Snack |
| 单价 | 6 coins |
| 购买数量 | 1 |
| 卡片更新 | `card_pet_food = seen`、`card_shop_buy = seen` |

购买后：

```text
coins -= 6
inventory.food_pet_snack += 1
next_event_id = event_feed_sunny
```

## 喂 Sunny

`Feed Sunny` 使用 `reward_rules.feed_pet`：

| 项 | 变化 |
|---|---|
| `inventory.food_pet_snack` | -1 |
| `pet.hunger` | -25 |
| `pet.happy` | +20 |
| `pet.fed_today` | true |
| `flags.sunny_first_snack_done` | true |
| `cards.card_d_dog_core` | played |
| `cards.card_pet_food` | played |
| `save_snapshot` | true |

宠物值必须限制在 0-100。

## 孩子端表达边界

孩子端只看到生活事件、奖励和收集反馈。以下词不得出现在孩子端标题、物品名或提示文案中：

```text
lesson
test
failed
word list
review
```

系统内部可以记录卡片状态、小游戏结果和进度，但界面表达必须保持 Home / Pet / Town / Shop 的生活冒险语境。
