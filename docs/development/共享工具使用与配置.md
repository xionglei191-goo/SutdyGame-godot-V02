# 共享工具使用与配置

> 适用项目：StudyGame V02  
> 目的：统一说明跨项目共享工具的路径、用法和 Git 约束。

## 1. 共享工具根目录

所有跨项目共享工具放在项目仓库外：

```text
/home/xionglei/GameProject/tools
```

当前共享工具：

```text
/home/xionglei/GameProject/tools/image_generator.js
/home/xionglei/GameProject/tools/godot_mcp/
```

这些工具不属于 V02 仓库，不应同步到 GitHub。

## 2. 本地配置模板

V02 提供路径配置模板：

```text
config/shared_tools_paths.env.example
```

需要本地 shell 配置时，可以复制为：

```bash
cp config/shared_tools_paths.env.example .env
```

`.env` 已被 `.gitignore` 忽略，不能提交。

## 3. 生图脚本

共享脚本路径：

```text
/home/xionglei/GameProject/tools/image_generator.js
```

调用示例：

```bash
node /home/xionglei/GameProject/tools/image_generator.js text "a warm storybook pet shop landmark for a children's English town game" /tmp/pet_shop.png 1024x1024
```

约束：

- 生图脚本可能包含本地凭据，只能保存在共享工具目录。
- 不要复制进 V02 仓库。
- 不要提交到 GitHub。
- 生成资产进入项目之前，必须保留 prompt 记录和用途说明。

## 4. Godot MCP 共享服务

共享目录：

```text
/home/xionglei/GameProject/tools/godot_mcp
```

目录结构：

```text
bridge/                 # Node.js MCP bridge
addon_template/          # Godot 插件模板 addons/godot_mcp
docs/Godot_MCP_服务说明.md
README_shared.md
```

### 4.1 给 V02 安装插件

当 V02 Godot 项目骨架创建后，将插件模板复制到项目内：

```bash
mkdir -p /home/xionglei/GameProject/SutdyGame-godot-V02/addons
cp -a /home/xionglei/GameProject/tools/godot_mcp/addon_template /home/xionglei/GameProject/SutdyGame-godot-V02/addons/godot_mcp
```

`addons/godot_mcp/` 已在 `.gitignore` 中忽略，因为 MCP 插件模板由共享目录维护。

### 4.2 启动 MCP bridge

```bash
cd /home/xionglei/GameProject/tools/godot_mcp/bridge
npm install
node mcp_godot_bridge.js
```

如果 `node_modules/` 已存在，通常不需要重新安装。

### 4.3 Godot 编辑器侧要求

使用 MCP 前确认：

1. Godot 编辑器已打开 V02 项目。
2. `addons/godot_mcp` 插件已在 Godot 插件面板启用。
3. 项目 autoload 中存在 MCP 插件注入的服务。
4. MCP bridge 使用共享目录中的 bridge 启动。

## 5. Git 约束

V02 `.gitignore` 已忽略：

```text
.godot/
tools/
local_tools/
.env
.env.*
addons/godot_mcp/
tmp/
logs/
*.log
export_presets.cfg
```

这保证本地凭据、共享工具副本、Godot 本地缓存和 MCP 插件副本不会被同步到 GitHub。

## 6. 后续维护规则

- 共享工具优先在 `/home/xionglei/GameProject/tools` 维护。
- 项目内只记录使用说明、路径模板和接入方法。
- 如果某个项目临时修改 MCP 插件，先在项目内验证，再决定是否同步回共享模板。
- 子 agent 需要使用共享工具时，必须在任务交接中说明调用路径和输出文件。
