# 世界结构与地图编辑架构

## 世界结构

V02 的世界以 Sunshine Town 为核心：

```text
Home
  -> Sunshine Town
  -> School
  -> Transport
  -> City / Nature / World
```

Home 是起点，不是菜单。Town 是主舞台。School 是重要区域，但不是产品脸面。Transport 是扩展世界的钥匙。

## 地图生产原则

V02 不应依赖一张完整大地图图片作为主地图。

推荐结构：

- 地面：TileMapLayer。
- 道路：TileMapLayer。
- 区域：District data。
- 建筑：Landmark Sprite。
- 点击区域：Hotspot data。
- 碰撞：Occupied cells。
- 交互点：Interaction cells。
- A-Z：Memory anchor layer。

## Godot 内编辑

V02 从一开始需要 Godot 内地图编辑能力。

推荐结构：

```text
WorldOverviewScene
  Runtime
    TileMapLayer_Ground
    TileMapLayer_Road
    LandmarkLayer
    InteractionLayer
    CollisionLayer

  EditorOnly
    Districts
    Roads
    Places
    MemoryAnchors
    CollisionCells
```

开发者在 EditorOnly 节点中拖地点、画道路、调区域、设交互点。工具同步写回数据。运行时只读数据生成地图。

## 为什么不把所有真数据只塞进 tscn

如果视觉位置在场景里，而 Quest、PlaceCard、A-Z、碰撞、交互点在数据里，容易出现画面和玩法错位。

因此最佳结构是：

```text
Godot 可视化编辑
  -> 写回数据
  -> 运行时读数据
  -> 测试校验数据合同
```

## 地图校验

保存地图时至少校验：

- place 是否在正确 district。
- interaction cell 是否不在 occupied cell。
- shop 是否连接道路。
- transport 是否连接主路。
- A-Z anchor 是否完整。
- quest focus hotspot 是否存在。
- 地标和热点矩形是否一致。
