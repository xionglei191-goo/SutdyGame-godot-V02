## 当前生活小镇内容生产基线

后续内容生产优先服务以下系统：

- NPC 问候与关系文本。
- 每日轻委托。
- 家具商品与商店轮换。
- 节日 / 天气 / 小镇轻事件。
- A-Z 场景故事和新词回访。

内容进入 runtime 前必须满足：

- 有稳定 ID。
- 可通过数据合同验证。

- 不依赖 Letter Snake、测试、课程表或单词背诵作为主线条件。

### NPC 问候与关系文本

NPC 内容生产目标是让 Sunshine Town 的居民、店主、宠物伙伴、故事伙伴和交通伙伴持续提供生活感回应。NPC 不是课程老师、考试引导、打卡监督或家长报告入口。

每个 NPC 内容包至少包含以下字段：

| 字段 | 要求 |
|---|---|
| `npc_id` | 稳定 ID，使用小写 `snake_case`，进入数据合同后除非有迁移记录不重命名。 |
| `display_name` | 孩子端显示名，中文优先，可带稳定英文名，例如 `Mina`、`Sunny`。 |
| `role` | 小镇身份与功能定位，例如居民朋友、商店店长、宠物伙伴、故事伙伴、交通伙伴。 |
| `daily_greetings` | 日常问候短句池，用于每日首次互动、天气或地点轻变化。 |
| `first_meeting` | 首次见面对白，说明关系和地点，不讲规则课。 |
| `relationship_stages` | 关系阶段反馈，至少区分初识、熟悉、亲近三个阶段。 |
| `request_dialogue` | 轻委托对白，包含接取、进行中、完成和无法完成时的温和反馈。 |
| `repeat_lines` | 重复互动短句，避免频繁触发同一句，也避免催促。 |
| `safety_notes` | 儿童安全说明，记录该 NPC 的禁用口吻、边界和特殊注意点。 |
| `visible_entry` | 孩子端可见入口或触发路径，例如 Town Plaza 靠近 Mina、Shop 入口、Bookshop、Bus Stop、Home/Sunny。 |
| `linked_request_ids` | 可选，关联轻委托 ID；没有委托时填写空列表并说明该 NPC 只提供氛围对白。 |
| `optional_anchor_links` | 可选，关联 A-Z anchor 或新词故事；只能辅助记忆，不能覆盖核心 anchor。 |

文本长度与语气规则：

- `daily_greetings`：每句中文 8-18 字为主，最多 24 字；可附 1 个 1-4 词英文环境短句，例如 `Good morning.`、`Thank you.`。
- `first_meeting`：1-2 句，每句中文 10-22 字；说明“我是谁 / 我在这里做什么 / 欢迎你来”，不解释学习系统。
- `relationship_stages`：每阶段 2-4 条短句；反馈应来自共同经历、地点回访、照顾 Sunny、帮忙收集等生活事件。
- `request_dialogue.accept`：1-2 句，说明生活目标和可见地点，例如“可以帮我找两根小树枝吗？”；不写成任务清单或课程要求。
- `request_dialogue.progress`：1 句，提醒可选继续；不使用“还没完成”“快一点”“今天必须”等压力表达。
- `request_dialogue.complete`：1-2 句，感谢、关系反馈和轻奖励提示；不把奖励写成成绩、评分或排名。
- `request_dialogue.repeat` / `repeat_lines`：每句中文 6-16 字，适合高频点击；避免同屏长段落。
- 英文只作为环境层、礼貌短句、物件名、地点标识、相册标签或语音片段。中文承担主要理解，英文不得成为通关门槛、答题要求或纠错内容。
- 所有对白应像小镇生活中的具体回应：天气、道路、商店、书、宠物、家具、资源和回访地点。避免抽象训导、学习目标、知识点讲解。

儿童安全文本边界：

- 禁止失败惩罚、倒扣资源、错过惩罚、连续登录压力和限时追赶口吻。
- 禁止学习压力、课程 / 考试 / 单词背诵 / 打卡 / 作业 / 排名 / 分数评价。
- 禁止家长报告口吻，例如“我要告诉爸爸妈妈”“家长会看到你的表现”“今天学习不够”。
- 禁止开放陌生人社交、私聊邀约、现实见面邀约、离开安全地点或隐瞒家长的暗示。
- 禁止羞辱、嘲笑、恐吓、威胁、失望凝视、道德绑架和亏欠感。
- 买不起、没收集到、重复互动、离开任务时，只给温和生活反馈，例如“没关系，晚点再来看看。”。
- NPC 不评价孩子聪明、笨、乖不乖或是否让大人满意，只回应具体行动和小镇关系。

新增 NPC 文本进入数据合同验证前，需要完成以下检查：

- `npc_id`、`request_id`、`anchor_id` 等引用必须稳定且唯一。
- 所有孩子端文本必须通过儿童安全扫描，拦截失败惩罚、学习压力、课程 / 考试 / 打卡、家长报告、开放陌生人社交、羞辱和恐吓关键词。
- 每个可互动 NPC 必须有 `visible_entry`，隐藏 contract 按钮或调试入口不能作为完成依据。
- 关联轻委托时，`linked_request_ids` 必须能追溯到目标、奖励、关系反馈、重复规则和可见操作入口。
- 可选 A-Z 关联必须使用 `optional_anchor_links`，不得重写、替换或移动核心 `core_anchor_id`；新增词仍需补齐 `letter`、`core_anchor_id`、`world_place_id`、`story_memory`、`visual_hook` 和 `review_path`。
- 英文短句必须能被中文上下文理解，不得要求孩子输入、选择或背诵后才能推进生活 RPG 主循环。

### 每日轻委托

每日轻委托内容包至少包含：

- 稳定 request ID。
- NPC。
- 目标物或目标动作。
- 奖励。
- 关系反馈。
- 当日去重与重复规则。
- 可见操作入口。

委托不惩罚缺席，不制造连续登录压力，不要求完成小游戏或学习测试。

#### V02-DESIGN-003 商店与家具轮换策划

商店服务生活 RPG / cozy town 的日常回访：孩子可以用金币购买家具、小装饰、宠物用品和少量资源材料，让 Home Room 更像自己的小屋，也让 Sunny 有稳定的照顾空间。商店不是学习奖励柜、课程兑换处、限时促销活动或打卡压力入口。

商品分类：

- `basic_furniture` 基础家具：小桌、地毯、椅子、收纳篮等，主要用于小屋摆放、划分生活角落和建立家园表达。

- `festival_weather` 节日 / 天气轻物件：雨滴贴纸、晴天风铃、微风飘带、小集市旗等，只作为氛围变体和轻回访理由，不绑定限时倒计时。

#### V02-DESIGN-005 节日与天气轻事件规划

轻事件服务 Sunshine Town 的每日氛围、居民回应和自然回访，不是活动任务链、限时运营、连续登录奖励或学习打卡。事件只改变今天的小镇表情：HUD 今日状态、NPC 问候、资源点细节、商店小角落、相册提示和 A-Z anchor 回访线索。孩子可以看见、路过、互动或完全忽略，系统都不记录失败、不补偿损失、不写“错过了”。

优先级说明：

- `P0 基础氛围`：适合先实现，只依赖今日状态、问候文案、基础资源刷新和少量 anchor 提示，例如晴天、微风、雨后。
- `P1 回访扩展`：增加集市、故事安静日等小镇节奏变化，连接每日轻委托和商店轮换，但不强制完成。
- `P2 节日轻变体`：儿童节 / 小镇庆祝日等低频氛围，不使用倒计时、限定损失或连续登录压力，商品和相册纪念可后续回流。

轮换与安全总规则：

- 每天最多 1 个主轻事件，必要时可叠加 1 个弱天气标签，例如“晴天 + 微风”，但孩子端只显示一句今日状态。
- P0 事件可高频自然轮换；P1 每周低频出现；P2 由季节、运营配置或本地日期触发，但不得显示剩余天数、倒计时或“最后机会”。
- 同一事件连续出现时只换问候和资源表现，不制造“第几天奖励”；缺席当天不会保留催促、补签或损失记录。
- 事件可提高某类每日轻委托的刷新权重，不能把委托变成强制条件；不完成委托也能逛商店、回 Home、看相册和回访 anchor。
- 事件商品只进入商店 `festival_weather` 或普通轮换角，不覆盖 P0 常驻商品；未购买不提示损失，后续同类天气或节日可回流。

## 14. 验收标准

内容基线通过标准：

- 每个首批事件都能追溯到 A-Z、0 基础补齐或 curriculum 来源。
- 首批卡片数量控制在约 20 张，不扩大到完整教材库。
- 首批场景围绕 Home/Pet/Town，不把 School 变成产品脸面。
- 首批小游戏只做 Letter Snake。
- 孩子端表达保持生活冒险语言，不出现 lesson/test/word list。
- 家长和系统内部可以记录学习状态，但孩子端只看到奖励和收集。
